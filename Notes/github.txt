～githubのターミナルで使用できるコマンド～
stage:
git add .
※これはすべてのファイルをステージする際に使用する（すべてではない場合はファイルを指定する必要がある）

commit:
git commit -m "ここにメッセージを書く"

push:
git push origin ブランチ名
ブランチの確認方法:
git branch

pull(データを持ってくる):
git pull origin ブランチ名

fetch(同期する):
git fetch

VSCで設定されているユーザーの名前とメアドを確認（変更）する方法
・名前
確認方法
git config --global user.name
変更方法
git config --global user.name "ここに変更したい名前を書く"

・メアド
確認方法
git config --global user.email
変更方法
git config --global user.email "ここに変更したいメアドを書く"

～エラーが出た時の対処法（その時はこれで解決できた）～
１・「Repository not found」
これはホストじゃなくて招待された側にその時起きて、
招待された人が承認をしていなかったことによってホームにリポジトリが追加されていなかった。
２・「Make sure you configure your ‘user.name’ and ‘user.email’」
まずそれらが設定されているかを確認して、設定されていなかったら設定する（やり方は上にかいてある）

～コミット名に関して～
feat: 新しい機能を追加した場合
新しい機能の追加やモジュールの実装を行ったときに使用します。
例: feat: add user authentication system
例: feat: implement multiplayer mode

fix: バグ修正に使用
既存のバグや不具合を修正した場合に使用します。
例: fix: resolve issue with card shuffle
例: fix: correct button alignment on mobile

docs: ドキュメント関連の変更
READMEやコメントなど、ドキュメントに関する変更を行ったときに使用します。
例: docs: update README with setup instructions
例: docs: add code comments to explain card logic

style: コードのフォーマットやスタイルの変更
機能に影響しないコードのスタイルやフォーマット（インデント、スペース、改行など）を変更したときに使用します。
例: style: apply consistent indentation in all files
例: style: update CSS class names to follow BEM convention

refactor: リファクタリング（機能変更なし）
機能を変更せずにコードの可読性や効率を改善したときに使用します。
例: refactor: simplify game loop logic
例: refactor: consolidate duplicate functions into a single utility

test: テストコードの追加や変更
テストコードの追加や修正を行ったときに使用します。
例: test: add unit tests for player actions
例: test: update test cases for card validation

chore: プロジェクトに直接影響しない変更
ビルドツールの設定変更や依存関係の更新など、プロジェクト全体に影響を与えるが、機能やバグ修正に関わらない変更を行ったときに使用します。
例: chore: update dependencies
例: chore: clean up unused files

perf: パフォーマンス改善
コードのパフォーマンス改善を行った場合に使用します（速度向上やメモリ削減など）。
例: perf: optimize card shuffle algorithm
例: perf: reduce image loading time by lazy loading

revert: 以前のコミットを取り消す
以前のコミットを取り消す場合に使用します（間違った変更を元に戻す）。
例: revert: undo changes to game logic
例: revert: remove experimental feature
