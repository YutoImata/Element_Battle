～githubのターミナルで使用できるコマンド～
stage:
git add .
※これはすべてのファイルをステージする際に使用する（すべてではない場合はファイルを指定する必要がある）

commit:
git commit -m "ここにメッセージを書く"

push:
git push origin ブランチ名
ブランチの確認方法:
git branch

pull(同期):
git pull origin ブランチ名


～コミット名に関して～
以下のようなものが主に使用される

feat
新しい機能を追加した場合に使用します。

fix
バグ修正に使用します。

docs
ドキュメント関連の変更に使用します（例: README、コメントなど）。

style
コードのフォーマットやスタイルの変更（機能に影響しないもの）に使用します。

refactor
機能の変更を伴わないリファクタリングに使用します（コードの改善など）。

test
テストコードやテスト関連の変更に使用します。

chore
プロジェクトや他の開発者に直接影響しない軽微な変更とみなされる時に使用する

perf
パフォーマンス改善に使用します

revert
以前のコミットを取り消す場合に使用します。
